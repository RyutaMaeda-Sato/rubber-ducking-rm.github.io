d:T3e70,<h2>はじめに</h2>
<p>クラウドサービスの見直しや統合などの理由で、AWSアカウントを削除する必要が生じることがあります。しかし、単にアカウントを閉鎖するだけでは不十分で、予期せぬ料金発生や情報漏洩のリスクが残る可能性があります。この記事では、AWSアカウントを安全かつ確実に削除するための手順を詳しく解説します。</p>
<h2>この記事で学べること</h2>
<ul>
<li>AWSアカウント削除前の準備と確認事項</li>
<li>全AWSリソースの安全な削除方法</li>
<li>アカウント削除の具体的な手順</li>
<li>削除後の注意点と確認事項</li>
</ul>
<h2>アカウント削除前の準備</h2>
<h3>データのバックアップ</h3>
<pre class="language-bash"><code class="language-bash code-highlight"><span class="code-line"><span class="token comment"># S3バケットの重要データをローカルにダウンロード</span>
</span><span class="code-line">aws s3 <span class="token function">sync</span> s3://your-important-bucket ./local-backup
</span><span class="code-line">
</span><span class="code-line"><span class="token comment"># RDSデータベースのスナップショットを作成（必要に応じて）</span>
</span><span class="code-line">aws rds create-db-snapshot <span class="token punctuation">\</span>
</span><span class="code-line">  --db-instance-identifier your-db-instance <span class="token punctuation">\</span>
</span><span class="code-line">  --db-snapshot-identifier final-snapshot
</span></code></pre>
<h3>請求情報の確認</h3>
<p>アカウント削除前に、以下の請求関連の確認を行いましょう：</p>
<ol>
<li><strong>未払い請求の確認</strong></li>
<li><strong>現在進行中の使用料の確認</strong></li>
<li><strong>将来的な予約インスタンスや前払いコミットメントの確認</strong></li>
</ol>
<h2>主要リソースの削除チェックリスト</h2>
<h3>コンピューティングリソース</h3>
<pre class="language-bash"><code class="language-bash code-highlight"><span class="code-line"><span class="token comment"># EC2インスタンスの一覧確認</span>
</span><span class="code-line">aws ec2 describe-instances
</span><span class="code-line">
</span><span class="code-line"><span class="token comment"># EC2インスタンスの終了</span>

</span><span class="code-line">
</span><span class="code-line"><span class="token comment"># Auto Scaling グループの削除</span>
</span><span class="code-line">aws autoscaling delete-auto-scaling-group --auto-scaling-group-name my-asg --force-delete
</span></code></pre>
<h3>ストレージリソース</h3>
<pre class="language-bash"><code class="language-bash code-highlight"><span class="code-line"><span class="token comment"># S3バケットの一覧確認</span>
</span><span class="code-line">aws s3 <span class="token function">ls</span>
</span><span class="code-line">
</span><span class="code-line"><span class="token comment"># S3バケットの削除（バケット内のオブジェクトも含めて）</span>

</span><span class="code-line">
</span><span class="code-line"><span class="token comment"># EBSボリュームの確認と削除</span>

</span><span class="code-line">aws ec2 delete-volume --volume-id vol-1234567890abcdef0
</span></code></pre>
<h3>データベースリソース</h3>
<pre class="language-bash"><code class="language-bash code-highlight"><span class="code-line"><span class="token comment"># RDSインスタンスの削除（最終スナップショットなし）</span>
</span><span class="code-line">aws rds delete-db-instance <span class="token punctuation">\</span>
</span><span class="code-line">  --db-instance-identifier your-db-instance <span class="token punctuation">\</span>
</span><span class="code-line">  --skip-final-snapshot
</span></code></pre>
<h3>ネットワークリソース</h3>
<pre class="language-bash"><code class="language-bash code-highlight"><span class="code-line"><span class="token comment"># Elastic IPの解放</span>

</span><span class="code-line"><span class="token comment"># VPCの削除</span>
</span><span class="code-line">aws ec2 delete-vpc --vpc-id vpc-12345678
</span></code></pre>

</span><span class="code-line"><span class="token comment"># IAMユーザーの削除</span>

</span><span class="code-line"><span class="token comment"># IAMロールの削除</span>

<h3>IAMポリシーの削除</h3>
<pre class="language-bash"><code class="language-bash code-highlight"><span class="code-line"><span class="token comment"># カスタムポリシーの一覧</span>

<h2>AWSアカウント削除の実行手順</h2>
<ol>
<li>
<p><strong>AWSマネジメントコンソールへのログイン</strong></p>
<ul>
<li>ルートユーザーでログインする必要があります</li>
<li>MFAが設定されている場合は認証を完了してください</li>
</ul>
</li>
<li>
<p><strong>アカウント設定ページへのアクセス</strong></p>
<ul>
<li>右上のアカウント名をクリック</li>
<li>ドロップダウンメニューから「アカウント」を選択</li>
</ul>
</li>
<li>
<p><strong>アカウント閉鎖セクションの操作</strong></p>
<ul>
<li>ページ下部の「アカウントの閉鎖」セクションまでスクロール</li>
<li>「アカウント閉鎖」ボタンをクリック</li>
</ul>
</li>
<li>
<p><strong>確認と実行</strong></p>
<ul>
<li>表示される注意事項を確認</li>
<li>チェックボックスにチェックを入れる</li>
<li>「アカウントを閉鎖」ボタンをクリック</li>
</ul>
</li>
</ol>
<h2>削除後の対応と注意点</h2>
<h3>最終請求の確認</h3>
<p>アカウント削除後も、最後の使用料に関する請求が発生します。登録されているメールアドレスと支払い方法が有効な状態を一定期間維持しましょう。</p>
<h3>復旧可能期間</h3>
<p>AWSアカウントは削除後90日間は復旧可能です。この期間中にアカウントへのアクセスが必要になった場合は、AWSサポートに連絡してください。</p>
<h3>Organizations関連の考慮事項</h3>
<p>AWS Organizationsのメンバーアカウントの場合、削除前に組織から脱退する必要があります：</p>
<pre class="language-bash"><code class="language-bash code-highlight"><span class="code-line"><span class="token comment"># Organizations からの脱退（メンバーアカウントの場合）</span>
</span><span class="code-line">aws organizations leave-organization
</span></code></pre>
<h2>トラブルシューティング</h2>
<h3>アカウント削除できない一般的な理由</h3>
<ol>
<li>
<p><strong>未削除のリソースが存在する</strong></p>
<ul>
<li>すべてのリージョンのリソースを確認してください</li>
<li>AWSリソースエクスプローラーを使用すると効率的です</li>
</ul>
</li>
<li>
<p><strong>AWS Organizationsの制約</strong></p>
<ul>
<li>管理アカウントは、すべてのメンバーアカウントを削除するまで閉鎖できません</li>
<li>メンバーアカウントは組織から脱退する必要があります</li>
</ul>
</li>
<li>
<p><strong>支払い問題</strong></p>
<ul>
<li>未払いの請求がある場合、アカウント削除が制限される場合があります</li>
</ul>
</li>
</ol>
<h2>ベストプラクティス</h2>
<ol>
<li>
<p><strong>リソース削除の自動化</strong></p>
<ul>
<li>大規模な環境では、AWS CDKやTerraformを使った削除も検討</li>
<li>定期的なリソースインベントリの取得習慣を持つ</li>
</ul>
</li>
<li>
<p><strong>複数リージョンの確認</strong></p>
<ul>
<li>リソースはグローバルに分散している可能性があるため、すべてのリージョンを確認</li>
</ul>
</li>
<li>
<p><strong>削除の証拠保存</strong></p>
<ul>
<li>コンプライアンスのために、削除完了の確認メールなどを保存</li>
</ul>
</li>
</ol>

<h2>まとめ</h2>
<p>AWSアカウントの削除は、単純なボタンクリックではなく、計画的なリソース削除とクリーンアップを必要とするプロセスです。この記事で解説した手順に従うことで、予期せぬ請求や情報漏洩のリスクを最小限に抑えつつ、安全にAWSアカウントを削除することができます。</p>
<p>重要なデータのバックアップを忘れずに行い、すべてのリソースが確実に削除されていることを複数回確認することをお勧めします。不明点がある場合は、AWSサポートに相談することも検討してください。</p>
