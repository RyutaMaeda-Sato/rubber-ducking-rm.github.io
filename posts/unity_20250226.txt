<h2>基本的なポーズ機能の仕組み</h2>
<p>Unityでゲームを一時停止するための基本的なメカニズムは、<code>Time.timeScale</code>プロパティを操作することです。このプロパティは、ゲーム内時間の進行速度を制御します：</p>
<ul>
<li><code>Time.timeScale = 1.0f</code> - 通常速度（デフォルト）</li>
<li><code>Time.timeScale = 0.0f</code> - 完全停止（ポーズ状態）</li>
<li><code>Time.timeScale = 0.5f</code> - 半分の速度（スローモーション）</li>
</ul>
<p>ポーズ状態では、物理演算、アニメーション、時間ベースの処理が停止しますが、入力検出やUIの操作は引き続き機能します。これにより、ポーズメニューのボタンを操作して、ゲームを再開したり終了したりすることが可能になります。</p>
<h2>実装手順</h2>
<h3>1. ポーズメニューUIの作成</h3>
<p>まず、ポーズメニュー用のUIを作成します：</p>
<ol>
<li>Hierarchyウィンドウで右クリック → UI → Canvas を選択してキャンバスを作成</li>
<li>作成したCanvasの子として、Panel（背景用）を追加</li>
<li>Panelの子として以下のUI要素を追加：
<ul>
<li>Text - "PAUSED"（タイトル）</li>
<li>Button - "RESUME"（再開ボタン）</li>
<li>Button - "QUIT"（終了ボタン）</li>
</ul>
</li>
</ol>
<p>ポーズメニューのCanvasは初期状態で非アクティブ（非表示）にしておきます。</p>

<h2>コード解説</h2>
<h3>重要なメソッド</h3>
<h4>TogglePause()</h4>
<p>このメソッドは、現在の状態（ポーズ中か通常プレイ中か）を切り替えます。ESCキーが押されたときに呼び出されます。</p>
<h4>PauseGame()</h4>
<p>ゲームを一時停止する主要な処理を行います：</p>
<ul>
<li><code>Time.timeScale = 0f</code>でゲーム内時間を停止</li>
<li>ポーズメニューUIを表示</li>
<li>ポーズ状態フラグを更新</li>
</ul>
<h4>ResumeGame()</h4>
<p>ゲームを再開する処理を行います：</p>
<ul>
<li><code>Time.timeScale = 1f</code>でゲーム内時間を通常に戻す</li>
<li>ポーズメニューUIを非表示</li>
<li>ポーズ状態フラグを更新</li>
</ul>
<h4>QuitGame()</h4>
<p>ゲームを終了する処理を行います：</p>
<ul>
<li>エディターの場合は再生モードを停止</li>
<li>ビルド済みアプリケーションの場合はアプリケーションを終了</li>
</ul>
<h2>UIデザインのポイント</h2>
<p>効果的なポーズメニューを設計するためのヒント：</p>
<ol>
<li><strong>シンプルさを保つ</strong> - 必要最小限の機能に絞り、視覚的にもシンプルに</li>
<li><strong>コントラスト</strong> - 背景とテキスト/ボタンのコントラストを高くし、視認性を確保</li>
<li><strong>半透明の背景</strong> - ゲーム画面が少し見えるよう、背景パネルは半透明にすると良い</li>
<li><strong>アニメーション</strong> - 表示/非表示時に軽いアニメーション効果を加えるとより洗練された印象に</li>
<li><strong>フォーカス管理</strong> - ポーズメニュー表示時に「再開」ボタンに自動的にフォーカスを設定</li>
</ol>

<h3>ポーズ中にもアニメーションが続く</h3>
<p><strong>問題</strong>: <code>Time.timeScale = 0f</code>を設定しても特定のアニメーションが停止しない。</p>
<p><strong>解決策</strong>: Animatorコンポーネントで「Update Mode」を「Normal」から「AnimatePhysics」に変更します。これにより、アニメーションがtime scaleの影響を受けるようになります。</p>
<h3>オーディオが停止しない</h3>
<p><strong>問題</strong>: ゲームを一時停止してもBGMや効果音が鳴り続ける。</p>
<p><strong>解決策</strong>: Time.timeScaleはオーディオには自動的に影響しないため、明示的に一時停止する必要があります：</p>
<pre class="language-csharp"><code class="language-csharp code-highlight"><span class="code-line"><span class="token punctuation">[</span><span class="token attribute"><span class="token class-name">SerializeField</span></span><span class="token punctuation">]</span> <span class="token keyword">private</span> <span class="token class-name">AudioSource<span class="token punctuation">[</span><span class="token punctuation">]</span></span> sourcesToPause<span class="token punctuation">;</span>
</span><span class="code-line">
</span><span class="code-line"><span class="token keyword">private</span> <span class="token return-type class-name"><span class="token keyword">void</span></span> <span class="token function">PauseGame</span><span class="token punctuation">(</span><span class="token punctuation">)</span>
</span><span class="code-line"><span class="token punctuation">{</span>
</span><span class="code-line">    <span class="token comment">// 既存のコード...</span>
</span><span class="code-line">
</span><span class="code-line">    <span class="token comment">// すべてのオーディオソースを一時停止</span>
</span><span class="code-line">    <span class="token keyword">foreach</span> <span class="token punctuation">(</span><span class="token class-name"><span class="token keyword">var</span></span> source <span class="token keyword">in</span> sourcesToPause<span class="token punctuation">)</span>
</span><span class="code-line">    <span class="token punctuation">{</span>
</span><span class="code-line">        source<span class="token punctuation">.</span><span class="token function">Pause</span><span class="token punctuation">(</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</span><span class="code-line">    <span class="token punctuation">}</span>
</span><span class="code-line"><span class="token punctuation">}</span>
</span><span class="code-line">
</span><span class="code-line"><span class="token keyword">private</span> <span class="token return-type class-name"><span class="token keyword">void</span></span> <span class="token function">ResumeGame</span><span class="token punctuation">(</span><span class="token punctuation">)</span>
</span><span class="code-line"><span class="token punctuation">{</span>
</span><span class="code-line">    <span class="token comment">// 既存のコード...</span>

</span><span class="code-line">    <span class="token keyword">foreach</span> <span class="token punctuation">(</span><span class="token class-name"><span class="token keyword">var</span></span> source <span class="token keyword">in</span> sourcesToPause<span class="token punctuation">)</span>

<pre class="language-csharp"><code class="language-csharp code-highlight"><span class="code-line"><span class="token punctuation">[</span><span class="token attribute"><span class="token class-name">SerializeField</span></span><span class="token punctuation">]</span> <span class="token keyword">private</span> <span class="token class-name">Button</span> pauseButton<span class="token punctuation">;</span>

<h2>まとめ</h2>
<p>プロフェッショナルなポーズメニューは、ゲームプレイ体験を大きく向上させる重要な要素です。基本的な実装は比較的シンプルですが、細部にこだわることで、ユーザーフレンドリーで直感的なポーズシステムを構築することができます。</p>
<p>この記事で紹介した手法を基本として、自分のゲームに最適なポーズ機能をカスタマイズしてください。ゲームのジャンルや対象プラットフォームによって、ポーズメニューに必要な機能は変わってくるでしょう。プレイヤーの視点に立ち、使いやすさを優先したデザインを心がけることが成功の鍵です。</p>
