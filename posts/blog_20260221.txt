d:T4ac6,<p>Claude Code を使うなら <code>CLAUDE.md</code> は必ず押さえておきたい仕組みです。この記事では、CLAUDE.md の役割・書き方・実際の活用パターンをまとめます。</p>
<h2>CLAUDE.md とは</h2>
<p><code>CLAUDE.md</code> は、Claude Code（Anthropic が提供する AI コーディングアシスタント）がリポジトリを開いたときに<strong>自動的に読み込む設定ファイル</strong>です。</p>
<p>プロジェクト固有の情報・規約・禁止事項などをここに書いておくと、Claude がそれを前提に動作してくれます。公式では「Claude's memory」とも呼ばれており、セッションをまたいで一貫したコンテキストを提供します。</p>
<h3>読み込まれるタイミング・ロケーション</h3>
<p>Claude Code を起動したとき、以下の場所にある CLAUDE.md が<strong>この順番で</strong>読み込まれます：</p>
<ol>
<li>ホームディレクトリ（<code>~/.claude/CLAUDE.md</code>）― 全プロジェクト共通のグローバル設定</li>
<li>リポジトリルート（<code>./CLAUDE.md</code>）― プロジェクト全体の設定（Git にコミットしてチームで共有）</li>
<li><code>.claude/CLAUDE.md</code>（<code>./CLAUDE.md</code> の代替として使えるサブディレクトリ配置）</li>
<li>サブディレクトリ（<code>./src/CLAUDE.md</code> など）― <strong>オンデマンド</strong>で読み込み（そのディレクトリ内のファイルを Claude が読んだときに初めて読み込まれる）</li>
</ol>
<p>また、<code>.claude/rules/</code> ディレクトリ内に置いた <code>.md</code> ファイルも自動的にプロジェクトメモリとして読み込まれます。ルールをファイル単位で分割管理したい場合に便利です。</p>
<blockquote>
<p><strong>注意</strong>: ファイル名は大文字・小文字を区別します。<code>CLAUDE.md</code>（CLAUDE は大文字、.md は小文字）でなければなりません。<code>claude.md</code> や <code>Claude.md</code> では認識されません。</p>
</blockquote>
<h2>何を書くべきか</h2>
<p>公式ドキュメントでは、以下の内容を書くことが推奨されています。</p>
<h3>1. プロジェクト概要</h3>
<pre class="language-markdown"><code class="language-markdown code-highlight"><span class="code-line"><span class="token title important"><span class="token punctuation">##</span> プロジェクト概要</span>
</span><span class="code-line">
</span><span class="code-line">個人ブログ — Next.js 14 (Static Export) + TypeScript + Tailwind CSS で構築。
</span><span class="code-line">記事は <span class="token code-snippet code keyword">`posts/*.md`</span> の Markdown ファイルで管理し、GitHub Pages にデプロイする。
</span></code></pre>
<p>Claude はプロジェクトの「何を作っているのか」を把握することで、的外れな提案を減らせます。</p>
<h3>2. 技術スタックと重要ファイル</h3>
<pre class="language-markdown"><code class="language-markdown code-highlight"><span class="code-line"><span class="token title important"><span class="token punctuation">##</span> 技術スタック</span>
</span><span class="code-line">
</span><span class="code-line"><span class="token list punctuation">-</span> フレームワーク: Next.js 14
</span><span class="code-line"><span class="token list punctuation">-</span> スタイリング: Tailwind CSS
</span><span class="code-line"><span class="token list punctuation">-</span> 記事形式: Markdown (gray-matter)
</span><span class="code-line">
</span><span class="code-line"><span class="token title important"><span class="token punctuation">##</span> 重要なファイル</span>
</span><span class="code-line">
</span><span class="code-line"><span class="token table"><span class="token table-header-row"><span class="token punctuation">|</span><span class="token table-header important"> パス </span><span class="token punctuation">|</span><span class="token table-header important"> 役割 </span><span class="token punctuation">|</span>
</span></span></span><span class="code-line"><span class="token table"><span class="token table-header-row"></span><span class="token table-line"><span class="token punctuation">|</span><span class="token punctuation">---</span><span class="token punctuation">|</span><span class="token punctuation">---</span><span class="token punctuation">|</span>
</span></span></span><span class="code-line"><span class="token table"><span class="token table-line"></span><span class="token table-data-rows"><span class="token punctuation">|</span><span class="token table-data"> <span class="token code-snippet code keyword">`src/app/lib/functions.ts`</span> </span><span class="token punctuation">|</span><span class="token table-data"> 記事取得・ページ生成のユーティリティ </span><span class="token punctuation">|</span>
</span></span></span><span class="code-line"><span class="token table"><span class="token table-data-rows"><span class="token punctuation">|</span><span class="token table-data"> <span class="token code-snippet code keyword">`src/components/`</span> </span><span class="token punctuation">|</span><span class="token table-data"> 共通コンポーネント </span><span class="token punctuation">|</span>
</span></span></span></code></pre>
<p>ファイル構成を教えておくと、「どこに何を書けばいいか」を Claude が自力で判断しやすくなります。</p>
<h3>3. よく使うコマンド</h3>
<pre class="language-markdown"><code class="language-markdown code-highlight"><span class="code-line"><span class="token title important"><span class="token punctuation">##</span> コマンド</span>
</span><span class="code-line">
</span><span class="code-line">\`\`\`bash
</span><span class="code-line">npm run dev        # 開発サーバー起動
</span><span class="code-line">npm run build      # ビルド
</span><span class="code-line">npm run lint       # ESLint チェック
</span><span class="code-line">\`\`\`
</span></code></pre>
<p>Claude はコマンドを直接実行する機能を持っているため、どのコマンドを使うべきかを明示しておくと効率的です。</p>
<h3>4. コーディング規約</h3>
<pre class="language-markdown"><code class="language-markdown code-highlight"><span class="code-line"><span class="token title important"><span class="token punctuation">##</span> コーディング規約</span>
</span><span class="code-line">
</span><span class="code-line"><span class="token list punctuation">-</span> TypeScript を使用する（型推論を活用し、明示的な型付けは必要な箇所のみ）
</span><span class="code-line"><span class="token list punctuation">-</span> コンポーネントは <span class="token code-snippet code keyword">`src/components/`</span> に配置する
</span><span class="code-line"><span class="token list punctuation">-</span> スタイルは Tailwind CSS クラスで記述する（インライン style は最小限）
</span><span class="code-line"><span class="token list punctuation">-</span> ダークモード対応: Tailwind の <span class="token code-snippet code keyword">`dark:`</span> プレフィックスを使用する
</span></code></pre>
<p>チームやプロジェクトで決めているルールを書いておくと、Claude が提案するコードがそのルールに沿ったものになります。</p>
<h3>5. 禁止事項・制約</h3>
<pre class="language-markdown"><code class="language-markdown code-highlight"><span class="code-line"><span class="token title important"><span class="token punctuation">##</span> 禁止事項</span>
</span><span class="code-line">
</span><span class="code-line"><span class="token list punctuation">-</span> <span class="token code-snippet code keyword">`node_modules/`</span> や <span class="token code-snippet code keyword">`out/`</span> 内のファイルを直接編集しない
</span><span class="code-line"><span class="token list punctuation">-</span> <span class="token code-snippet code keyword">`npm run build`</span> 前に <span class="token code-snippet code keyword">`out/`</span> を手動削除しない
</span><span class="code-line"><span class="token list punctuation">-</span> 本番コミットに <span class="token code-snippet code keyword">`.env`</span> ファイルを含めない
</span></code></pre>
<p>「やってはいけないこと」を明記することで、Claude の誤操作を防ぎます。</p>
<h2>実際の書き方のポイント</h2>
<h3>簡潔に書く</h3>
<p>CLAUDE.md はコンテキストウィンドウに入るため、<strong>長すぎると他の情報を圧迫</strong>します。200行を目安に、箇条書きや表を活用してコンパクトにまとめましょう。</p>
<h3>「なぜ」より「何を」を優先する</h3>
<p>理由よりも、Claude に取ってほしい行動・避けてほしい行動を具体的に書きます。</p>
<pre class="language-markdown"><code class="language-markdown code-highlight"><span class="code-line"><span class="token title important"><span class="token punctuation">#</span> 良い例</span>
</span><span class="code-line"><span class="token list punctuation">-</span> コンポーネントは <span class="token code-snippet code keyword">`src/components/`</span> に配置する
</span><span class="code-line">
</span><span class="code-line"><span class="token title important"><span class="token punctuation">#</span> 悪い例</span>
</span><span class="code-line"><span class="token list punctuation">-</span> コンポーネントの配置場所には歴史的な経緯があり、過去に何度かリファクタリングを...
</span></code></pre>
<h3>動的な情報は入れない</h3>
<p>日付やバージョン番号のような変化しやすい情報は、古くなったときに混乱の元になります。「最新のドキュメントを参照してください」のように書くほうが安全です。</p>
<h2>業務での使いどころ</h2>
<h3>チーム開発での共有設定</h3>
<p>CLAUDE.md をリポジトリにコミットしておくと、チーム全員が同じコンテキストで Claude を使えます。オンボーディング資料としても機能します。</p>
<pre><code class="code-highlight"><span class="code-line">プロジェクトルート
</span><span class="code-line">├── CLAUDE.md          ← チーム共通設定（コミット対象）
</span><span class="code-line">├── src/
</span><span class="code-line">│   ├── CLAUDE.md      ← フロントエンド固有のルール
</span><span class="code-line">│   └── api/
</span><span class="code-line">│       └── CLAUDE.md  ← API レイヤー固有のルール
</span></code></pre>
<h3>コードレビューの基準を伝える</h3>
<p>「このプロジェクトでは XXX パターンを使う」「YYY ライブラリは禁止」などの方針を書いておけば、Claude が提案するコードがレビュー基準に沿ったものになります。</p>
<h3>スキル・カスタムコマンドの説明</h3>
<p>Claude Code は <code>/</code> から始まるカスタムコマンド（スキル）を <code>.claude/commands/</code> に定義できます。CLAUDE.md にその説明を書いておくと、どのコマンドがあるかを Claude が把握できます。</p>
<pre class="language-markdown"><code class="language-markdown code-highlight"><span class="code-line"><span class="token title important"><span class="token punctuation">##</span> カスタムコマンド</span>
</span><span class="code-line">
</span><span class="code-line"><span class="token list punctuation">-</span> <span class="token code-snippet code keyword">`/new-post`</span> : 新しいブログ記事を作成する
</span><span class="code-line"><span class="token list punctuation">-</span> <span class="token code-snippet code keyword">`/check-post`</span> : 記事のファクトチェックとレビューを行う
</span></code></pre>
<h2>ハマりやすいポイント</h2>
<h3>ファイルが長すぎて無視される</h3>
<p>CLAUDE.md が長すぎると、後半の内容がコンテキストウィンドウに収まらず無視されることがあります。<strong>重要な情報は上部に書く</strong>ことを意識し、詳細は別ファイルに分けて CLAUDE.md からリンクする方法が有効です。</p>
<pre class="language-markdown"><code class="language-markdown code-highlight"><span class="code-line"><span class="token title important"><span class="token punctuation">##</span> 詳細ドキュメント</span>
</span><span class="code-line">
</span><span class="code-line"><span class="token list punctuation">-</span> コーディング規約の詳細: <span class="token url">[<span class="token content">docs/coding-rules.md</span>](<span class="token url">docs/coding-rules.md</span>)</span>
</span><span class="code-line"><span class="token list punctuation">-</span> デプロイ手順: <span class="token url">[<span class="token content">docs/deploy.md</span>](<span class="token url">docs/deploy.md</span>)</span>
</span></code></pre>
<h3>グローバルとプロジェクトの設定が衝突する</h3>
<p><code>~/.claude/CLAUDE.md</code>（グローバル）に書いたルールとプロジェクトの CLAUDE.md のルールが矛盾すると、Claude が混乱します。グローバルには本当に全プロジェクト共通のことだけを書き、プロジェクト固有のことはプロジェクトの CLAUDE.md に書きましょう。</p>
<h3>設定を書いても Claude が従わない場合</h3>
<p>指示が曖昧だと Claude が解釈を誤ることがあります。「推奨する」より「必ず〜する」「〜してはいけない」のように<strong>断定的な表現</strong>を使うほうが効果的です。</p>
<p>また、Claude Code の設定ファイル（<code>.claude/settings.json</code>）で <code>permissions</code> を設定することで、ファイルへのアクセス制限などをより確実に制御できます。</p>
<h2><code>/init</code> コマンドで自動生成する</h2>
<p>CLAUDE.md を一から書くのが面倒な場合は、Claude Code のターミナルで以下のコマンドを実行します。</p>
<pre class="language-bash"><code class="language-bash code-highlight"><span class="code-line">/init
</span></code></pre>
<p>Claude がリポジトリを解析して、プロジェクト概要・ディレクトリ構造・使用技術などを盛り込んだ CLAUDE.md の初期テンプレートを自動生成してくれます。生成後に不要な箇所を削ったり、禁止事項を追記したりするのが現実的なワークフローです。</p>
