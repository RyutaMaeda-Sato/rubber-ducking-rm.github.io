<h2>unified エコシステムとは</h2>
<p><a href="https://unifiedjs.com/" rel="nofollow" target="_blank">unified</a> は、<strong>テキスト処理を AST ベースのパイプラインで行う</strong>ためのフレームワーク。
Markdown・HTML・自然言語など異なる形式を統一した仕組みで変換できる。</p>
<pre><code class="code-highlight"><span class="code-line">入力テキスト
</span><span class="code-line">    ↓  parse（パーサー）
</span><span class="code-line">   AST（抽象構文木）
</span><span class="code-line">    ↓  transform（プラグイン群）
</span><span class="code-line">   変換後 AST
</span><span class="code-line">    ↓  stringify（シリアライザー）
</span><span class="code-line">出力テキスト
</span></code></pre>
<p><code>remark</code> は Markdown 用の unified プロセッサ、<code>rehype</code> は HTML 用の unified プロセッサで、両者を橋渡しする <code>remark-rehype</code> を介することで Markdown → HTML の変換チェーンが成立する。</p>

<pre><code class="code-highlight"><span class="code-line">Markdown テキスト (string)

<h2>ハマりやすいポイント</h2>
<h3>1. allowDangerousHtml と rehypeRaw のセット忘れ</h3>
<p><code>remarkRehype</code> に <code>allowDangerousHtml: true</code> を渡しただけでは生 HTML は残らない。
<code>rehypeRaw</code> を後続に追加して初めて hast ノードに変換される。片方だけ設定しても効果がない。</p>
<h3>2. プラグインの順序</h3>
<p><code>rehypePrism</code> は hast 上のコードブロックを処理するため、<code>remarkRehype</code> より後に置く必要がある。
また <code>rehypeRaw</code> は <code>remarkRehype</code> の直後に置くのが安全（raw ノードが他のプラグインで処理される前に解決しておく）。</p>
<pre class="language-typescript"><code class="language-typescript code-highlight"><span class="code-line"><span class="token comment">// NG: rehypeRaw を後ろに置きすぎると raw ノードが別プラグインに触られる場合がある</span>
</span><span class="code-line"><span class="token punctuation">.</span><span class="token function">use</span><span class="token punctuation">(</span>remarkRehype<span class="token punctuation">,</span> <span class="token punctuation">{</span> allowDangerousHtml<span class="token operator">:</span> <span class="token boolean">true</span> <span class="token punctuation">}</span><span class="token punctuation">)</span>
</span><span class="code-line"><span class="token punctuation">.</span><span class="token function">use</span><span class="token punctuation">(</span>rehypePrism<span class="token punctuation">)</span>   <span class="token comment">// ← raw ノードが残ったまま処理される</span>
</span><span class="code-line"><span class="token punctuation">.</span><span class="token function">use</span><span class="token punctuation">(</span>rehypeRaw<span class="token punctuation">)</span>
</span><span class="code-line">
</span><span class="code-line"><span class="token comment">// OK: rehypeRaw は remarkRehype の直後</span>
</span><span class="code-line"><span class="token punctuation">.</span><span class="token function">use</span><span class="token punctuation">(</span>remarkRehype<span class="token punctuation">,</span> <span class="token punctuation">{</span> allowDangerousHtml<span class="token operator">:</span> <span class="token boolean">true</span> <span class="token punctuation">}</span><span class="token punctuation">)</span>
</span><span class="code-line"><span class="token punctuation">.</span><span class="token function">use</span><span class="token punctuation">(</span>rehypeRaw<span class="token punctuation">)</span>     <span class="token comment">// ← まず raw ノードを解決</span>
</span><span class="code-line"><span class="token punctuation">.</span><span class="token function">use</span><span class="token punctuation">(</span>rehypePrism<span class="token punctuation">)</span>
</span></code></pre>
<h3>3. rehype-prism-plus のバージョンと CSS の用意</h3>
<p><code>rehype-prism-plus</code> v2 はクラスを付与するだけで、CSS は自分で用意する必要がある。
Prism.js のテーマ CSS（<code>prism-tomorrow.css</code> など）を <code>layout.tsx</code> や <code>global.css</code> でインポートしないとハイライトが見えない。</p>
<pre class="language-typescript"><code class="language-typescript code-highlight"><span class="code-line"><span class="token comment">// src/app/layout.tsx</span>
</span><span class="code-line"><span class="token keyword">import</span> <span class="token string">"prismjs/themes/prism-tomorrow.css"</span><span class="token punctuation">;</span>
</span></code></pre>
<h3>4. ESM 専用パッケージ</h3>
<p><code>unified</code> v11 以降、<code>remark-*</code> / <code>rehype-*</code> の主要パッケージはすべて <strong>ESM 専用</strong>になっている。
<code>require()</code> で呼び出す CommonJS 環境では動作しない。Next.js 13+ の App Router はデフォルトで ESM を扱えるため問題ないが、カスタムスクリプト（<code>scripts/</code> など）を CommonJS で書いている場合は注意が必要。</p>
