<h2>特徴まとめ</h2>





























<table><thead><tr><th>項目</th><th>内容</th></tr></thead><tbody><tr><td>対応バージョン</td><td>React 19.2 以上</td></tr><tr><td>主な用途</td><td>エフェクト内の「反応しなくていいロジック」を切り出す</td></tr><tr><td>最新値の参照</td><td>常に最新の props / state を参照可能</td></tr><tr><td>依存配列への追加</td><td>不要（追加するとエラー）</td></tr><tr><td>Lint サポート</td><td><code>eslint-plugin-react-hooks v6</code> 以上が必要</td></tr></tbody></table>
<h2>注意点：依存配列を隠す用途には使わない</h2>
<p>エフェクトの再実行トリガーになるべき値まで <code>useEffectEvent</code> に移すと、変化に反応すべきタイミングを見逃してバグになります。</p>
<pre class="language-jsx"><code class="language-jsx code-highlight"><span class="code-line"><span class="token comment">// ❌ 悪い例：docId が変わってもタイマーがリセットされない</span>
</span><span class="code-line"><span class="token keyword">const</span> onSave <span class="token operator">=</span> <span class="token function">useEffectEvent</span><span class="token punctuation">(</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token arrow operator">=></span> <span class="token punctuation">{</span>
</span><span class="code-line">  <span class="token function">saveDoc</span><span class="token punctuation">(</span>docId<span class="token punctuation">,</span> content<span class="token punctuation">)</span><span class="token punctuation">;</span>
</span><span class="code-line"><span class="token punctuation">}</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</span><span class="code-line">
</span><span class="code-line"><span class="token function">useEffect</span><span class="token punctuation">(</span><span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token arrow operator">=></span> <span class="token punctuation">{</span>
</span><span class="code-line">  <span class="token comment">// docId 切り替え時にクリーンアップが走らない</span>
</span><span class="code-line">  <span class="token comment">// → 旧ドキュメントの保存完了コールバックや状態リセットが必要な場合に問題になる</span>
</span><span class="code-line">  <span class="token keyword">const</span> id <span class="token operator">=</span> <span class="token function">setInterval</span><span class="token punctuation">(</span>onSave<span class="token punctuation">,</span> <span class="token number">30000</span><span class="token punctuation">)</span><span class="token punctuation">;</span>
</span><span class="code-line">  <span class="token keyword control-flow">return</span> <span class="token punctuation">(</span><span class="token punctuation">)</span> <span class="token arrow operator">=></span> <span class="token function">clearInterval</span><span class="token punctuation">(</span>id<span class="token punctuation">)</span><span class="token punctuation">;</span>
</span><span class="code-line"><span class="token punctuation">}</span><span class="token punctuation">,</span> <span class="token punctuation">[</span><span class="token punctuation">]</span><span class="token punctuation">)</span><span class="token punctuation">;</span> <span class="token comment">// docId を依存配列に含めていない</span>
</span></code></pre>
<p><code>useEffectEvent</code> は常に最新の <code>docId</code> を参照するため、「保存先が古いドキュメントになる」ことはありません。しかし依存配列が <code>[]</code> のままでは、<code>docId</code> 切り替え時にクリーンアップ処理が走らず、旧ドキュメントに紐づく後処理（状態リセット・コールバック制御など）が必要な場面でバグになります。</p>
<p>「エフェクトをいつ走らせるかに関係する値」は依存配列に残し、「エフェクト内で参照するだけの値」を <code>useEffectEvent</code> に移すのが正しい使い方です。</p>
<h2>まとめ</h2>
<p><code>useEffectEvent</code> が解決する本質は、<strong>エフェクトの「起動条件」と「処理内容」を分けること</strong>にあります。</p>
<ul>
<li>スクロール監視の開始条件は <code>articleId</code>、でも送信データには <code>userId</code> も必要</li>
<li>タイマーのリセット条件は <code>docId</code>、でも保存内容には最新の <code>content</code> が必要</li>
</ul>
<p>こういった「エフェクトのトリガーにはしたくないけど、最新値を参照したい」場面に絞って使うと、依存配列がシンプルになり、意図の伝わるコードになります。</p>
