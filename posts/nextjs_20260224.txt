d:T8f46,<p>このブログでも使っている <code>gray-matter</code> は、Markdownファイルの先頭に書くYAMLヘッダー（frontmatter）をJavaScriptオブジェクトとして取り出すライブラリだ。Next.jsの静的ブログを作る際に必ずと言っていいほど登場するので、基本的な使い方から型安全な活用法まで整理する。</p>
<h2>frontmatterとは</h2>
<p>Markdownファイルの先頭に <code>---</code> で囲んで書くYAML形式のメタデータ領域を「frontmatter」と呼ぶ。</p>
<pre class="language-markdown"><code class="language-markdown code-highlight"><span class="code-line"><span class="token front-matter-block"><span class="token punctuation">---</span>
</span></span><span class="code-line"><span class="token front-matter-block"><span class="token front-matter yaml language-yaml"><span class="token key atrule">title</span><span class="token punctuation">:</span> <span class="token string">"記事タイトル"</span>
</span></span></span><span class="code-line"><span class="token front-matter-block"><span class="token front-matter yaml language-yaml"><span class="token key atrule">date</span><span class="token punctuation">:</span> <span class="token string">"2026-02-24"</span>
</span></span></span><span class="code-line"><span class="token front-matter-block"><span class="token front-matter yaml language-yaml"><span class="token key atrule">tags</span><span class="token punctuation">:</span> <span class="token punctuation">[</span><span class="token string">"Next.js"</span><span class="token punctuation">,</span> <span class="token string">"TypeScript"</span><span class="token punctuation">]</span></span>
</span></span><span class="code-line"><span class="token front-matter-block"><span class="token punctuation">---</span></span>
</span><span class="code-line">
</span><span class="code-line">ここから本文が始まる。
</span></code></pre>
<p><code>---</code> より前の部分が frontmatter、それ以降が Markdown 本文だ。<code>gray-matter</code> はこの2つを分離して返してくれる。</p>

<h2>frontmatterの設計指針</h2>
<p>このブログのフロントマター設計をもとに、実用的な設計のポイントをまとめる。</p>
<p><strong>必須フィールドは最小限に</strong></p>
<p><code>title</code> と <code>date</code> だけを必須とし、あとは省略可能にすることで記事作成のハードルを下げられる。</p>
<pre class="language-yaml"><code class="language-yaml code-highlight"><span class="code-line"><span class="token punctuation">---</span>
</span><span class="code-line"><span class="token key atrule">title</span><span class="token punctuation">:</span> <span class="token string">"記事タイトル"</span>     <span class="token comment"># 必須</span>
</span><span class="code-line"><span class="token key atrule">date</span><span class="token punctuation">:</span> <span class="token string">"2026-02-24"</span>       <span class="token comment"># 必須</span>
</span><span class="code-line"><span class="token punctuation">---</span>
</span></code></pre>
<p><strong><code>description</code> は省略可能にして自動補完</strong></p>
<p><code>description</code> がない場合、本文の冒頭から自動生成する <code>generateExcerpt</code> 関数を用意しておくと、毎回書く手間が省ける。</p>
<pre class="language-typescript"><code class="language-typescript code-highlight"><span class="code-line"><span class="token keyword">const</span> description<span class="token operator">:</span> <span class="token builtin">string</span> <span class="token operator">=</span> data<span class="token punctuation">.</span>description <span class="token operator">??</span> <span class="token function">generateExcerpt</span><span class="token punctuation">(</span>content<span class="token punctuation">)</span><span class="token punctuation">;</span>
</span></code></pre>
<p><strong>null許容フィールドにはデフォルト値を</strong></p>
<p><code>tags</code> が <code>null</code> のままだと <code>.map()</code> でエラーになるので、読み込み時に空配列に変換しておく。</p>
<pre class="language-typescript"><code class="language-typescript code-highlight"><span class="code-line">tags<span class="token operator">:</span> data<span class="token punctuation">.</span>tags <span class="token operator">||</span> <span class="token punctuation">[</span><span class="token punctuation">]</span><span class="token punctuation">,</span>
</span></code></pre>
<h2>ハマりやすいポイント</h2>
<p><strong>YAML の配列記法</strong></p>
<p>frontmatter の配列は以下の2通りどちらでも書けるが、gray-matter は両方正しくパースしてくれる。</p>
<pre class="language-yaml"><code class="language-yaml code-highlight"><span class="code-line"><span class="token comment"># インライン記法</span>
</span><span class="code-line"><span class="token key atrule">tags</span><span class="token punctuation">:</span> <span class="token punctuation">[</span><span class="token string">"Next.js"</span><span class="token punctuation">,</span> <span class="token string">"TypeScript"</span><span class="token punctuation">]</span>
</span><span class="code-line">
</span><span class="code-line"><span class="token comment"># ブロック記法</span>
</span><span class="code-line"><span class="token key atrule">tags</span><span class="token punctuation">:</span>
</span><span class="code-line">  <span class="token punctuation">-</span> Next.js
</span><span class="code-line">  <span class="token punctuation">-</span> TypeScript
</span></code></pre>
<p><strong>日付はダブルクォートで囲む</strong></p>
<p><code>date: 2026-02-24</code> と書くと、YAML パーサーが <code>Date</code> オブジェクトとして解釈してしまう場合がある。文字列として扱いたいなら必ずクォートする。</p>
<pre class="language-yaml"><code class="language-yaml code-highlight"><span class="code-line"><span class="token key atrule">date</span><span class="token punctuation">:</span> <span class="token string">"2026-02-24"</span>  <span class="token comment"># 文字列として扱われる</span>
</span><span class="code-line"><span class="token key atrule">date</span><span class="token punctuation">:</span> <span class="token datetime number">2026-02-24</span>    <span class="token comment"># Date オブジェクトになることがある</span>
</span></code></pre>
<p><strong><code>content</code> にはfrontmatterが含まれない</strong></p>
<p><code>matter()</code> が返す <code>content</code> には frontmatter 部分（<code>---</code> で囲まれた領域）は含まれない。独自に <code>---</code> を除去する処理を書く必要はない。</p>
<h2>関連ツールとの比較</h2>
<p>frontmatter をパースするライブラリは gray-matter 以外にもある。</p>

























<table><thead><tr><th>ライブラリ</th><th>特徴</th><th>向いているケース</th></tr></thead><tbody><tr><td><code>gray-matter</code></td><td>YAML/JSON/TOML/CoffeeScript対応。デリミタカスタマイズ可。型定義内蔵</td><td>Next.js・Gatsby・Astroなどフレームワーク問わずに使いたいとき</td></tr><tr><td><code>front-matter</code></td><td>YAML専用、非常にシンプル</td><td>小規模スクリプトや YAML しか使わない場合</td></tr><tr><td><code>remark-frontmatter</code></td><td>remarkパイプラインに統合できる</td><td>unifiedパイプラインの中でfrontmatterを処理したいとき</td></tr></tbody></table>
<p>Next.js や Astro の静的ブログなら <code>gray-matter</code> が事実上の標準で、エコシステムのサンプルコードも最も多い。<code>remark-frontmatter</code> はパイプライン内でfrontmatterをAST上で扱えるが、単純にメタデータをオブジェクトとして取り出したいだけなら gray-matter の方がシンプルだ。</p>
<h2>まとめ</h2>
<ul>
<li><code>gray-matter</code> は <code>matter(fileContents)</code> の1行で frontmatter と本文を分離できる</li>
<li>TypeScript では <code>matter.GrayMatterFile&#x3C;string> &#x26; { data: YourType }</code> でキャストして型安全にする</li>
<li><code>description</code> の自動補完や <code>tags || []</code> のデフォルト値など、読み込み時に欠損を吸収しておくのが実装を楽にするコツ</li>
<li>日付フィールドは必ずクォートして文字列として扱う</li>
<li>シンプルなメタデータ取得なら gray-matter、パイプライン統合なら remark-frontmatter と使い分ける</li>
